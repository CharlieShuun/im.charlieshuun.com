在學術與實務中找尋平衡，致力於電子工程與網路安全的深度探索。

個人特質

身為一名高中生，我對技術的熱忱源於對「運作原理」的渴望。在面對 Python 程式碼中刺眼的紅色錯誤訊息（Error Code）時，我體悟到寫程式的本質並非追求一次成功，而是在不斷的 Debug 中磨練耐性，學會利用社群資源與翻譯工具定位問題，從中建立起嚴謹的邏輯推演能力。

我擁有一種「職人式」的專注，這點在硬體維修中尤為明顯。處理精密電路時，我學會了在微小空間內保持情緒穩定與手部精確度。這種特質讓我不再畏懼未知的挑戰，無論是軟體邏輯的斷點，還是硬體電路的故障，我都能以系統性的思維，在挫折中梳理出通往解決方案的路徑。

個人特長

雲端架構與虛擬化： 熟練管理 QTS 伺服器系統，並能運用 Docker 容器技術進行服務部署。我理解環境隔離（Isolation）的重要性，能獨立配置資源分配與監控系統健康狀態。

網路安全與傳輸： 掌握 WireGuard VPN 隧道加密技術，深入理解通訊埠轉發（Port Forwarding）與網路層安全策略，具備初步的防火牆日誌分析與威脅防禦能力。

硬體工程與診斷： 具備行動裝置的深度拆解能力，掌握刀頭烙鐵解焊與精密零件更換技術（如 Type-C 接口、主板組裝），並始終堅持「硬體完整性是資安第一道防線」的原則。

技術邏輯與錯誤排除：具備優秀的 Debugging 邏輯思維。在面對 Python 代碼報錯或系統配置衝突時，擅長利用翻譯資源與全球技術社群定位核心問題，透過不斷的測試與調優找出解決路徑。

實作專案

家用伺服器資安實務：我將 NAS 打造為多功能的家用核心，不僅架設了高效能的 Minecraft 伺服器，更透過升級 16GB 記憶體與網路調優，解決了高負載下的延遲問題。更重要的是，我實作了 WireGuard VPN 加密連結，確保資料在公共網路傳輸時的隱私安全。

行動裝置硬體解構專案：面對損壞的電子裝置，我獨立執行了完整的維修流程。從螢幕分離、斷開電源保護、到針對 Type-C 接口進行精準焊接。這項專案不僅恢復了裝置功能，更讓我深刻理解電子產品內部的空間佈局與零件協作原理。

Python 邏輯啟蒙路徑：我從零開始探索程式世界，經歷了從簡單的輸出指令到複雜邏輯判斷的轉變。透過分析錯誤代碼定位 Bug，我學會了如何自主規劃學習進度，並將抽象的演算法邏輯應用於實際問題的解決。

未來展望

步入大學後，我計畫將高中三年的實作經驗轉化為更深厚的理論基礎。我渴望進入電資領域，進一步探索嵌入式系統與物聯網（IoT）的結合。在資訊安全威脅日益嚴峻的當下，我希望利用我對硬體構造與網路架構的理解，研發出更具韌性的防禦體系。

我對技術的追求不僅限於個人成長。我期許未來能參與開源社群，將我的實作筆記與心得分享給更多技術愛好者。我深信，一名優秀的工程師不僅要能解決當下的 Bug，更要具備洞察未來技術趨勢的眼光，並始終保持對世界的好奇與對技術的敬畏。

常見問題

Q1：為什麼你的歷程中較少看到大型資訊競賽或證照？ ▾
比起在考場中追求標準答案，我更傾向於在現實環境中解決問題。我將時間投入於從零開始構建系統——例如從硬體維修、NAS 伺服器部署到 Docker 虛擬化架構。對我而言，解決一個真實存在的網路攻擊或硬體故障，比獲得一張證照更能磨練我的能力。 我相信需要的不僅是解題者，更是能守護實體架構的實踐者。
Q2：你提到的「硬體維修」經驗，對資管系資安組有什麼幫助？ ▾
資訊安全不應僅止於軟體層面。透過拆解平板與焊接電路，我深刻理解了硬體端的物理完整性（Hardware Integrity）是安全的第一道防線。這種從底層向上構建的思維，讓我能更全面地評估系統漏洞，而不僅僅是依賴軟體補丁。
Q3：你在學習 Python 過程中提到「翻譯錯誤代碼」，這代表你的程式能力尚在起步嗎？ ▾
那是我的啟蒙階段，我刻意記錄下那個過程，是為了提醒自己面對未知的耐性。現在的我已能透過分析 Container Logs 進行服務排錯。我認為資安人員最重要的特質不是天生會寫代碼，而是當系統報錯時，能冷靜地定位問題並解決它的能力。
Q4：為什麼選擇元智資管資安組？而非單純的資工系？ ▾
我熱愛技術，但我更在乎技術如何被應用於保護資訊價值。元智資管強調『學用合一』，與我喜歡動手架設伺服器、解決真實網路問題的性格完全契合。我希望在這裡能將我的實作經驗系統化，成為具備管理思維的資安專家。
